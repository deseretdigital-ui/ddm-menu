Requirements

  • Push the page over.
  • Smooth (enough) animation.
  • Disable vertical scrolling when open.
  • Disable horizontal scrolling when open.
  • Maintain page scroll position through opening and closing.
  • Must play well with fixed position elements on screen.
  • Reset scroll within the menu to top when opening a second time.
  • Click outside to close (clicks should not trigger anything on the page, just close the menu).
  • Menu scrolls independent of page.
  • Smooth scrolling.
  • Needs to be easy to pop back into page
  • Option to position menu on the right
  • Multiple menus on same page operate independently
  • Option to teardown menu at any point

Issues

  • Answer backward compatibility questions.
  • Technical annoyance: Header, Menus, and content, as well as other fixed position elements, animated separately, rather than the animation being controlled on a single parent element.
  • Android 2.3.5 Stock Browser: Menu not scrollable (But Android Firefox on Android works perfect; don't have a clue how to fix this without getting complicated (css-transform translate instead of overflow auto/hidden)).
  • Mobile Safari: Address bar opens when the menu opens.
  • Desktop Browsers (Firefox, Chrome, Safari, IE8): Page is scrollable.
  • Mobile Safari: Scroll bounce on page after scrolling past end of menu.
  • Background color on menu doesn't extend into overscroll area.
  • Scrollbar detached from menu.
  • Overlay on menu open to catch clicks.
  • Public interface doesn't match current ddm-menu version (function call, event names, methods, namespaces).
